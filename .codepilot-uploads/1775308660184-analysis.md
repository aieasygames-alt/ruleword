## 二、网络请求架构

### 2.1 HTTP 封装层次

```
HttpRequest (Cocos通用层, 支持GET/POST, 防重复, M-Token鉴权)
  └─ SdkHttp (SDK层, request/requestv3/request$/request2$/requestv4)
      └─ 业务层 (SdkConfig/SdkStorage/GameReport/RankManager)
          └─ 第三方SDK (DataNexus/GravityEngine/转化归因)
```

### 2.2 启动流程时序

```
SmSdk.init()
  → 加载平台SDK分包 (sdkwx)
  → 初始化 GravityEngine 分包

SmSdk.load()
  → initConfig()
    → GET  api.sm0.fun/v2/cfg/{gameid}/{version}/sdk
    → GET  api.sm0.fun/v2/cfg/{gameid}/{version}/sdk_shield
    → GET  cdwaterbear.cn/ip/queryex (IP查询，判断屏蔽)
  → wx.login() → code
    → GET  api.sm0.fun/v2/code2openid (code换openid)
  → POST dc.sm0.fun/v2/userdata/getUserData (加载云存档)
  → RankManager.init()
  → 进入游戏
```

- **总关卡数**: 100关 (ID: 70001-70100)
- **最小网格**: 8x10 (80格, 6箭头) — 第1关
- **最大网格**: 41x53 (2173格, 158箭头) — 第3关
- **箭头数量范围**: 6 ~ 196
- **主要难度梯度**: 网格从 20x20 逐步增大到 30x50+
