# Arrow Puzzle（箭头解谜）游戏开发方案

## Context

用户希望开发一款类似《箭了又箭》的箭头解谜小游戏，上架到 ruleword.com 平台。ruleword.com 是一个基于 **Astro SSG + Cloudflare** 的免费在线益智游戏平台，现有 111 款浏览器游戏，支持 8 种语言。游戏需要自包含（纯前端），通过 localStorage 保存进度，适配桌面和移动端。

## 目标平台适配

| 约束 | 要求 |
|------|------|
| 框架 | Astro SSG 静态页面 + 自包含客户端 JS |
| URL | `ruleword.com/games/arrow-puzzle/` |
| 存储 | localStorage 保存游戏进度 |
| 设备 | 桌面 + 平板 + 手机响应式 |
| 语言 | 至少支持中英文，最好 8 语言 |
| 部署 | Cloudflare 静态托管 |

## 技术选型

- **渲染**: HTML5 Canvas（高性能，适合网格渲染和滑动动画）
- **语言**: TypeScript → 编译为纯 JavaScript
- **构建**: Vite（打包为单个 JS 模块嵌入 Astro 页面）
- **样式**: CSS-in-JS / 内联样式（无需外部依赖）
- **零依赖**: 不使用任何第三方游戏引擎或 UI 库

## 核心游戏机制

### 基础规则
1. 网格上放置多个箭头，每个箭头有方向（上/下/左/右）
2. 玩家**点击**箭头 → 箭头沿指向方向**直线滑行**直到离开网格
3. 若路径被其他箭头或障碍物**阻挡** → 箭头**弹回原位**，计一次失误
4. 所有箭头全部清出网格 → **通关**
5. 失误次数达上限（3次） → **失败**，可重试

### 难度四层递进

| 层级 | 关卡范围 | 新机制 | 核心挑战 |
|------|---------|--------|---------|
| 第1层 | 1-30 | 单方向箭头，小网格(4×4~5×5) | 找正确消除顺序 |
| 第2层 | 31-70 | 多方向箭头(2~4向)，中网格(6×6) | 起手顺序影响叠加收益 |
| 第3层 | 71-120 | 大网格(7×7~8×8)，密集箭头 | 维持每步消除中位数 |
| 第4层 | 121-200 | 数字方块障碍 | "避免最小损失"策略 |

### 特殊元素
- **多方向箭头**: 点击后向多个方向同时射出，需所有方向路径畅通
- **数字方块**: 显示数字 N，被箭头经过 N 次后消除；增加路径规划深度
- **墙壁/障碍物**: 静态阻挡，箭头无法通过

## 关卡生成算法

采用 **反向插入算法** 保证每个关卡必定有解：

```
function generateLevel(difficulty):
  1. 创建空网格
  2. 根据难度确定箭头数量 N
  3. 循环 N 次：
     a. 随机选择一个网格边缘和进入方向
     b. 沿该方向向内扫描，找到第一个空位
     c. 放置箭头（方向为进入方向的反方向）
     d. 记录插入顺序
  4. 解 = 插入顺序的反序
  5. 验证解的唯一性/难度，不满足则重新生成
```

手工关卡用于教学和关键体验节点（约40个），其余由算法生成。

## 视觉设计

- **背景**: 深色(#1a1a2e)，高对比度
- **网格线**: 低透明度灰色，辅助定位
- **箭头颜色**: 按方向分色（红↑/蓝↓/绿←/黄→）
- **动画**: 箭头滑出（ease-out 300ms）、弹回（shake 动画）、消除（缩放淡出）
- **UI**: 极简风格，与 ruleword.com 现有游戏一致
- **无障碍**: 支持 `prefers-reduced-motion`

## 开发阶段

### 阶段1：核心引擎（第1-3天）
1. 搭建项目脚手架（Vite + TypeScript）
2. 实现 Grid 数据模型与 Arrow 实体
3. 实现 Canvas 渲染引擎（网格、箭头绘制）
4. 实现点击交互与碰撞检测
5. 实现箭头滑动动画
6. 实现基础游戏流程（开始→玩→胜/负）

### 阶段2：关卡系统（第4-6天）
1. 设计关卡数据格式
2. 手工制作 40 个教学关卡
3. 实现反向插入生成算法
4. 批量生成 160 个程序关卡
5. 实现关卡选择界面
6. 实现关卡进度保存

### 阶段3：进阶机制（第7-9天）
1. 实现多方向箭头机制
2. 实现数字方块机制
3. 实现 Undo/撤销系统
4. 实现提示系统（BFS 求解器）
5. 实现星级评分系统

### 阶段4：体验打磨（第10-12天）
1. 音效系统（Web Audio API 合成音）
2. 动画优化（弹回shake、消除粒子、通关庆祝）
3. 触摸手势（缩放、平移大网格）
4. 响应式适配（手机竖屏优先）
5. 国际化（中英文）

### 阶段5：集成上线（第13-14天）
1. 编写 Astro 页面模板
2. 编写 SEO 游戏指南内容
3. Vite 打包 → 嵌入 Astro
4. 浏览器兼容性测试（Chrome/Safari/Firefox）
5. 性能优化（Canvas 渲染、内存管理）
6. 提交 ruleword.com 上架
